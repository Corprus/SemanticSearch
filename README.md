# SemanticSearch

## Name
SemanticSearch

## Description
Семантический поиск по пользовательским документам с использованием модели ML.

## Ипользуемая ML модель

PyPi Sentence transformers (https://pypi.org/project/sentence-transformers/), all-MiniLM-L6-v2
Модель расчитана на расчет эмбеддингов.

## Архитектура проекта
Проект состит из нескольких модулей, экспортируеыъ в docker, и объединённых в docker compose

### Модуль app
Основной модуль программы (бэк). Обрабатывает запросы от фронта, работает с сущностями в БД Postgres (используется SQLAlchemy). Публичные команды экспортируются как REST-серис через FastAPI

### Модуль worker
Модуль работы с ML (бэк). Принимает команды, принятые через RabbitMQ от App. Может быть распараллелен или запущен в несколько инстанций.Генерирует и сохраняет embedding вектора от сущностей, принятых из app.

### Модуль ui
Фронт для сервиса. Написан на streamlit, позволяет работать с бэком.
Для обычного пользователя предусмотрены основные действия:
1. Создание и авторизация пользователя
2. Добавление документов пользователя в систему. (с фоновым процессингом)
3. Поиск от имени пользователя по документам, с использованием семантического поиска.
4. Добавление кредитов (используются для индексации добавленных документов и при поисках)
5. Просмотр истории поиска
6. Просмотр истории транзакций
7. Для пользователя с правами администратора предусмотрена админка.

### Тесты
Добавлены юнит-тесты для проекта.
Необходимые зависимости прописаны в tests/requirements.txt
Пользоваться:
Для базовых (модульных) тестов:
pytest -c pytest-modules.ini 

Для тестов системы в целом (end-to-end):
pytest -c pytest-endtoend.ini

## Общий функционал сервиса

Пользователь может загружать документы, будет (за кредиты) производиться их индексация, и искать с использованием семантики по ним (за поиск тоже будут сниматься кредиты).
Цель: удобный семантический поисковик по загруженным текстовым документам пользователя

## Как пользоваться
Сервис запускаетс с помощью команды docker compose up.
Сервис запускается на http://localhost
на http://localhost/api находится бэк, можно использовать swagger
Остальные порты закрыты (работает внутри сети докера)

По умолчанию создаются 2 пользователя с соответствующими правами:
demo / demo123
admin / admin123

Им генерятся документы и история. И так при каждом запуске (если не хотите перегенерить - выключаете drop_all)
Дальше поднимается UI на localhost, на http://localhost/api - висит уже само приложение (то есть сваггер на localhost/api/docs)
